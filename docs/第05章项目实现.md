## 第5章. 项目实现 基于CAF构建键盘应用

现在我们重新创建一个新的空白项目，项目名称为keyboard，并创建一个空白的BSP包，开发板名称为atguigu\_mini\_keyboard，vendor填写atguigu。按照上一章的示例，启用基本CAF框架，并创建main module。

## 1. 整体项目架构

### 1.1. 按键与HID数据流

```
flowchart LR
    key_scan[按键采集]
    mode_switch[模式拨档]
    encoder_scan[旋钮采集]

    keyboard_core((键盘核心))
    host_protocol[上位机协议]
    send_sched[发送调度]

    usb_host[USB主机]
    usb_input[USB输入]

    ble_host[BLE主机]
    ble_input[BLE输入]

    %% 输入采集到键盘核心
    key_scan -- 按键事件 --> keyboard_core
    mode_switch -- 模式事件 --> keyboard_core
    encoder_scan -- 旋钮事件 --> keyboard_core

    %% 模式切换影响发送调度
    mode_switch -- 模式事件 --> send_sched

    %% HID报告进入发送调度
    keyboard_core -- HID报告事件 --> send_sched

    %% 功能键上报到上位机协议
    keyboard_core -- 功能键事件 --> host_protocol

    %% USB通道
    send_sched -- 发送事件\(USB\) --> usb_input
    usb_input -- 链路状态 / 已发送 --> send_sched
    usb_host -- HID输入输出 --> usb_input
    usb_input -- HID输入输出 --> usb_host

    %% BLE通道
    send_sched -- 发送事件\(BLE\) --> ble_input
    ble_input -- 链路状态 / 已发送 --> send_sched
    ble_host -- HID输入输出 --> ble_input
    ble_input -- HID输入输出 --> ble_host

    %% 协议切换 / 状态反馈
    send_sched -- 协议切换 / 状态 --> keyboard_core
    keyboard_core -- 协议切换 / 状态 --> ble_input
```

## 2. 矩阵按键

[CAF: Buttons module](https://docs.nordicsemi.com/bundle/ncs-3.2.3/page/nrf/libraries/caf/buttons.html)

Nordic在NCS中附带了很多现成的module，也就是说如果我们构建一个键盘应用，并不用一切都从0开始。例如矩阵按键和GPIO按键扫描程序button\_module。Button module基于GPIO + 中断扫描矩阵按键，所以我们需要打开GPIO硬件。

### 2.1 需求描述

实现一个 **6 行 4 列矩阵键盘**，但实际只使用其中 **18 个有效按键**。当用户按下有效按键时，系统可以识别对应的按键位置，并通过 RTT 输出对应的 key\_id。

### 2.2 主控GPIO

```
行：
R0 -> P0.15
R1 -> P0.07
R2 -> P0.12
R3 -> P0.04
R4 -> P1.09
R5 -> P0.08

列：
C0 -> P0.05
C1 -> P0.06
C2 -> P0.26
C3 -> P0.30
```

```c
row-gpios = <&gpio0 15 (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
			    <&gpio0 7 (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
			    <&gpio0 12 (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
			    <&gpio0 4 (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
			    <&gpio1 9 (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
			    <&gpio0 8 (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>;
		col-gpios = <&gpio0 5 GPIO_ACTIVE_HIGH>,
			    <&gpio0 6 GPIO_ACTIVE_HIGH>,
			    <&gpio0 26 GPIO_ACTIVE_HIGH>,
			    <&gpio0 30 GPIO_ACTIVE_HIGH>;
```

### 2.3 涉及CAF模块

```
app_event_manager
CAF button_event
CAF keep_alive_event
CAF power_event
```


## 4. 编码器模块

### 4.1 需求描述

实现一个 **EC11 旋钮编码器旋转识别功能**。当用户顺时针或逆时针旋转旋钮时，系统可以识别旋转方向、旋转步数和累计角度，并通过 RTT 输出旋钮方向、当前步数和累计角度。

### 4.2. 主控GPIO

```plain
EC11A -> P0.10
EC11B -> P1.06
```

```c
encoder: encoder {
    compatible = "gpio-qdec";
    status = "okay";

    /* EC11A -> P0.10, EC11B -> P1.06. Swap order if direction is reversed. */
    gpios = <&gpio0 10 (GPIO_PULL_UP | GPIO_ACTIVE_HIGH)>,
    <&gpio1 6 (GPIO_PULL_UP | GPIO_ACTIVE_HIGH)>;
    steps-per-period = <2>;
    zephyr,axis = <INPUT_REL_WHEEL>;
    sample-time-us = <2000>;
    idle-poll-time-us = <2000>;
    idle-timeout-ms = <200>;
};
```

### 4.3 涉及CAF模块

```c
Zephyr gpio-qdec
Zephyr input subsystem
CAF keep_alive_event
CAF power_event
app_event_manager
```

### 4.4 参考提示词

```
请帮我实现一个基于 Zephyr 官方 gpio-qdec 的 EC11 旋钮编码器模块。

要求：
- A/B 两相接在 P0.10 和 P1.06
- 两个输入都配置 GPIO_PULL_UP | GPIO_ACTIVE_HIGH
- 在 DTS 中启用 gpio-qdec
- steps-per-period 配置为 2
- zephyr,axis 设置为 INPUT_REL_WHEEL
- sample-time-us 为 2000
- idle-poll-time-us 为 2000
- idle-timeout-ms 为 200
- 应用层监听 INPUT_EV_REL 和 INPUT_REL_WHEEL
- value > 0 视为 clockwise，value < 0 视为 counterclockwise
- 每次旋转打印方向、步数、累计角度
- 旋钮旋转时也要复用低功耗逻辑，低功耗时发送 wake_up_event，正常时发送 keep_alive_event
- 因为 P0.10 是 NFC 复用脚，DTS 里必须加 &uicr { nfct-pins-as-gpios; };
```

## 5. 电源管理模块

### 5.1 需求描述

电源管理模块应该包含充放电状态汇报，IP5306保活，电池电压测量以及SOC水平汇报。这里面我们需要驱动IP5306，但是Zephyr中没有现成的驱动，所以我们需要套用Zephyr的开发标准，开发一套IP5306的驱动

1. 通过 I2C 读充电状态
2. 定期（每8s）给 IP5306 的 wakeup 脚打 keepalive 脉冲
3. 读取

### 5.2 IP5306驱动开发

https://docs.zephyrproject.org/latest/develop/modules.html

我们使用外部zephyr module的形式开发驱动，所以要遵循zephyr module约定的结构，大致如下：

### 5.3 主控GPIO引脚

```plain
- I2C SDA：P1.00
- I2C SCL：P0.24
- IP5306 wakeup/keepalive 引脚：P0.22，active low
- BAT_ADC: P0.31 / AIN7
- BAT_ADC_EN P0.09
```

### 5.4 参考提示词

```c
请在这个 Zephyr/NCS 项目中实现 IP5306 PMIC 驱动和电池状态模块。

硬件信息：
- MCU：nRF52840
- PMIC：IP5306，挂在 I2C0，地址 0x75
- I2C SDA：P1.00
- I2C SCL：P0.24
- IP5306 wakeup/keepalive 引脚：P0.22，active low
- 电池电压通过 /vbatt voltage-divider 采样

功能要求：
1. 新增 out-of-tree IP5306 驱动。
2. 驱动通过 I2C 读取 IP5306 状态：
   - 读寄存器 0x70 bit3 判断 charging
   - 读寄存器 0x71 bit3 判断 full
3. 驱动提供接口：
   - ip5306_init()
   - ip5306_get_status()
4. 驱动支持 keepalive 脉冲，防止 IP5306 轻载自动关机：
   - wakeup GPIO 每 8000 ms 产生一次有效脉冲
   - 脉冲宽度 500 ms
   - wakeup GPIO 是 active low，所以实际有效电平为低
5. 优先使用硬件 keepalive：
   - RTC2 计时
   - GPIOTE 控制 GPIO toggle
   - GPPI/PPI 连接 RTC compare 事件到 GPIOTE task
   - 尽量不依赖线程定时器，保证低功耗状态下仍能产生脉冲
6. 在 devicetree 增加 binding 和 board DTS 节点。
7. 在 battery_module 中初始化 IP5306，并周期性读取 charging/full 状态。
8. 保持现有项目风格，不引入不必要的重构。
    3300 mV -> 0%
4200 mV -> 100%
中间按 10% 档位四舍五入

验收标准：
- 项目能编译通过。
- devicetree 中有 compatible = "injoinic,ip5306" 的节点。
- IP5306 I2C 地址为 0x75。
- P0.22 每 8 秒出现一个约 500 ms 的低电平脉冲。
- battery_module 能读取并发布 charging/full 状态。
- 轻载待机时 IP5306 不会自动关断系统供电。
```

## 9. HID 流控与传输策略

### 目标

让键盘核心不关心当前走 USB 还是 BLE，由策略模块和流控模块统一决定。

### 需要提供的资料

- 三种模式下 HID 走向
- 是否允许 USB 和 BLE 同时在线
- 键盘报告是否覆盖旧值
- 音量等 consumer pulse 是否必须 FIFO

### 当前策略

- USB 模式：HID 走 USB
- BLE 模式：HID 走 BLE，普通 BLE profile
- 2.4G 模式：HID 走 BLE，dongle profile

### 涉及模块

- `mode_policy_module.c`
- `transport_policy_event`
- `hid_flowctrl_module.c`
- `keyboard_core_module.c`

### 验收标准

- 模式切换后不会向旧通道继续发按键
- 切换模式时会释放旧通道按键
- Consumer pulse 不丢失或乱序
- 键盘 report 不因拥塞积压旧状态

## 10. LED 灯效模块需求

### 10.1 功能目标

控制 WS2812 灯带，支持按键触发的 key fade 效果、主题色更新和灯带开关。

### 10.2 硬件

- LED strip：WS2812 over SPI。
- SPI1 SCK：P1.04。
- SPI1 MOSI/Data：P0.20。
- chain length：17。
- supply GPIO：P0.13 active high。

### 10.3 输入事件

- `button_event`：按键触发灯效。
- `theme_rgb_update_event`：更新主题色。
- `led_strip_en_event`：启停灯带。
- 生命周期和电源事件。

### 10.4 行为

- 按键按下时，在对应 LED 上触发渐隐效果。
- 主题色变化后，后续灯效使用新颜色。
- power down 时关闭或暂停灯带。
- wake up 后恢复。

### 10.5 验收

- 按键时对应灯位有渐隐效果。
- 上位机或设置页改主题后灯效颜色变化。
- 灯带关闭后不再发光。

## 11. UI 显示

### 11.1 功能目标

基于 LVGL + ST7789V 显示主界面和设置页面。

### 11.2 硬件

**所用 MCU 引脚（nRF52840）**

| 信号 | 外设 | GPIO 引脚 | 电平有效 | 说明 |
| --- | --- | --- | --- | --- |
| SCK | SPI3 | **P1.13** | — | SPI 时钟 |
| MOSI | SPI3 | **P0.28** | — | SPI 数据输出（只写，无 MISO） |
| CS | SPI3 | **P0.02** | ACTIVE\_LOW | 片选 |
| DC | GPIO | **P0.03** | ACTIVE\_HIGH | 数据/命令选择 |
| RESET | GPIO | **P1.10** | ACTIVE\_LOW | 硬件复位 |
| 背光 | PWM0 CH0 | **P1.11** | 反转极性 | 100Hz PWM，通过 `led_on`/`led_off` API 控制 |

**屏幕特性：**

| 参数 | 值 |
| --- | --- |
| 驱动芯片 | **Sitronix ST7789V** |
| 接口类型 | **SPI 4 线**（MIPI DBI Type C option 3，只写模式） |
| 最大 SPI 频率 | **32 MHz** |
| 色深 | **16 bit/pixel (RGB565)**，65K 色 |
| 物理分辨率 | 320 × 240 |
| 有效区域 | **320 × 172**（y 偏移 34，即顶部跳过 34 行） |
| x 偏移 | 0 |
| LVGL 像素格式 | 16 bit，启用字节交换 (`LV_COLOR_16_SWAP`) |
| 就绪等待时间 | 120 ms |

### 11.4 主界面显示

### 11.5 验收

- 电池状态变化时 UI 电量/充电图标变化。
- BLE 连接状态变化时 UI 更新。
