## 矩阵按键方案：蓝牙小键盘

| 项目    | 方案                                          |
| ----- | ------------------------------------------- |
| 按键结构  | 6 行 × 4 列矩阵                                 |
| 实际按键  | 18 个有效按键                                    |
| 主控    | nRF52840                                    |
| 驱动方案  | Zephyr `gpio-kbd-matrix`                    |
| 坐标映射  | Zephyr `input-keymap`                       |
| 上层事件  | 转换为 CAF `button_event`                      |
| 防鬼键   | 每个按键串联 1N4148W 二极管                          |
| 低功耗唤醒 | 按键事件触发 `keep_alive_event` / `wake_up_event` |

项目实现文档明确要求使用 **6x4 矩阵键盘**，实际有效按键为 **18 个**，并最终转换为 CAF `button_event`。 原理图中也能看到矩阵网络包含 `ROW0~ROW5`、`COL0~COL3`，并且按键部分使用 1N4148W 二极管。

## GPIO 分配

| 类型   |  编号 | nRF52840 引脚 |
| ---- | --: | ----------- |
| ROW0 | 行 0 | P0.15       |
| ROW1 | 行 1 | P0.07       |
| ROW2 | 行 2 | P0.12       |
| ROW3 | 行 3 | P0.04       |
| ROW4 | 行 4 | P1.09       |
| ROW5 | 行 5 | P0.08       |
| COL0 | 列 0 | P0.05       |
| COL1 | 列 1 | P0.06       |
| COL2 | 列 2 | P0.26       |
| COL3 | 列 3 | P0.30       |

## 设备树核心配置

```dts
kbd_matrix: kbd-matrix {
    compatible = "gpio-kbd-matrix";
    row-gpios = <&gpio0 15 (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
                <&gpio0 7  (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
                <&gpio0 12 (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
                <&gpio0 4  (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
                <&gpio1 9  (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>,
                <&gpio0 8  (GPIO_PULL_DOWN | GPIO_ACTIVE_HIGH)>;

    col-gpios = <&gpio0 5 GPIO_ACTIVE_HIGH>,
                <&gpio0 6 GPIO_ACTIVE_HIGH>,
                <&gpio0 26 GPIO_ACTIVE_HIGH>,
                <&gpio0 30 GPIO_ACTIVE_HIGH>;

    actual-key-mask = <0x3e 0x3e 0x1e 0x2b>;
    no-ghostkey-check;

    keymap {
      compatible = "input-keymap";
      keymap = <
        MATRIX_KEY(0, 3, INPUT_KEY_ESC)

        MATRIX_KEY(1, 0, INPUT_KEY_NUMLOCK)
        MATRIX_KEY(1, 1, INPUT_KEY_KPSLASH)
        MATRIX_KEY(1, 2, INPUT_KEY_KPASTERISK)
        MATRIX_KEY(1, 3, INPUT_KEY_KPMINUS)

        MATRIX_KEY(2, 0, INPUT_KEY_KP7)
        MATRIX_KEY(2, 1, INPUT_KEY_KP8)
        MATRIX_KEY(2, 2, INPUT_KEY_KP9)

        MATRIX_KEY(3, 0, INPUT_KEY_KP4)
        MATRIX_KEY(3, 1, INPUT_KEY_KP5)
        MATRIX_KEY(3, 2, INPUT_KEY_KP6)
        MATRIX_KEY(3, 3, INPUT_KEY_KPPLUS)

        MATRIX_KEY(4, 0, INPUT_KEY_KP1)
        MATRIX_KEY(4, 1, INPUT_KEY_KP2)
        MATRIX_KEY(4, 2, INPUT_KEY_KP3)

        MATRIX_KEY(5, 0, INPUT_KEY_KP0)
        MATRIX_KEY(5, 1, INPUT_KEY_KPDOT)
        MATRIX_KEY(5, 3, INPUT_KEY_KPENTER)
      >;
      row-size = <6>;
      col-size = <4>;
    };
};
```

## 软件数据流

```text
矩阵 GPIO
  ↓
gpio-kbd-matrix 扫描
  ↓
input-keymap 坐标映射
  ↓
CAF button_event
  ↓
keyboard_core
  ↓
HID Report
  ↓
USB HID / BLE HID
```

## 设计要点

* **扫描策略**：列线逐列拉高，读取行线电平，确定 `(row, col)`。
* **去抖处理**：由 Zephyr input / matrix 驱动层处理，避免机械轴抖动导致重复触发。
* **有效键过滤**：通过 `actual-key-mask = <0x3e 0x3e 0x1e 0x2b>` 屏蔽不存在的矩阵点。该属性按列配置，每个 bit 表示该列实际存在的行。
* **事件解耦**：矩阵模块只负责产生按键事件，不直接生成 HID 报告。
* **低功耗适配**：按键事件同时可作为系统保活或唤醒入口。
* **面试重点**：解释矩阵扫描、二极管防鬼键、设备树硬件抽象、CAF 事件驱动解耦，以及 USB/BLE HID 共用输入源的架构价值。

## 当前工程接入计划

当前仓库已经存在 `input`、`keymap`、`hid`、`mode`、`transport` 目录。矩阵键盘接入时不需要再新建一套并行框架，应沿现有骨架推进。

建议数据边界如下：

| 阶段 | 输入 | 输出 | 落地位置 |
| --- | --- | --- | --- |
| 矩阵扫描 | GPIO 行列电平 | Zephyr input 事件 | board DTS / `prj.conf` |
| 坐标映射 | row / col | 内部 key id | `input-keymap` / `input_manager.c` |
| 键义映射 | key id | HID usage | `keymap.c` |
| 报告维护 | HID usage + pressed | 6KRO report | `hid_report.c` |
| 链路发送 | HID report | USB/BLE report | `transport_usb.c` / `transport_ble.c` |

## 设备树落地清单

在 `boards/mingDev/mini-keyboard/mini-keyboard.dts` 中应确认：

- `gpio0` 和 `gpio1` 节点状态可用。
- `ROW0~ROW5` 对应 P0.15、P0.07、P0.12、P0.04、P1.09、P0.08。
- `COL0~COL3` 对应 P0.05、P0.06、P0.26、P0.30。
- 行列方向与二极管方向匹配。
- `actual-key-mask` 与 18 个实际焊接按键一致：COL0 为 row1~5，COL1 为 row1~5，COL2 为 row1~4，COL3 为 row0、row1、row3、row5。
- `input-keymap` 覆盖全部有效键，不为无效矩阵点分配业务键值。

如果扫描方向与硬件二极管方向不一致，现象通常不是“完全不能用”，而是部分组合键异常、鬼键过滤失效或低功耗唤醒不稳定。因此这一步必须用实测事件逐键验证。

## 18 键建议键位语义

第一阶段建议先按标准数字小键盘闭环，不急着做复杂宏：

| 功能组 | 建议键 |
| --- | --- |
| 数字输入 | Num 0~9 |
| 运算符 | `/`、`*`、`-`、`+`、`.` |
| 控制键 | Enter、Backspace 或 Num Lock |
| 系统键 | 模式键 / Fn 组合键 |

若实际结构只有 18 个键，应优先保证数字输入、Enter、退格、模式切换和常用运算符。Num Lock 可以通过 Fn 组合键实现，避免占用独立按键。

## input 到 HID 的转换规则

建议内部按键事件保持极简：

```c
struct key_event {
  uint16_t key_id;
  bool pressed;
  uint32_t timestamp_ms;
};
```

转换规则：

1. input 层只产生 `key_id + pressed`。
2. keymap 层把 `key_id` 转成 HID usage 或特殊动作。
3. mode key / Fn key 属于特殊动作，不进入普通键盘 report。
4. 普通键调用 `hid_keyboard_report_add_key()` 或 `hid_keyboard_report_remove_key()`。
5. 每次 report 变化后，由当前 `mode_manager_get_mode()` 决定发送到哪条 transport。

## 调试与验收方法

矩阵调试建议按从底到上的顺序进行：

1. 打开 Zephyr input 日志，确认每个有效物理键都有事件。
2. 验证无效矩阵点不会产生事件。
3. 快速点击同一个键，确认没有明显抖动重复。
4. 同时按下多个键，确认二极管防鬼键有效。
5. 将每个 key id 打印出来，和实际键帽位置做表格核对。
6. 接入 keymap 后，再验证 HID usage 是否符合预期。
7. 接入 transport 后，分别验证 USB 和 BLE 输出一致。

## 风险点

- `actual-key-mask` 写错会导致有效键丢失或无效点产生事件。
- 行列 GPIO 顺序写反会导致 keymap 全部错位。
- 二极管方向与扫描方向不匹配会导致组合键异常。
- P0.26、P0.30 等引脚需要确认是否被 NFC、低频晶振、调试或其他外设占用。
- 如果模式键也在矩阵中，长按识别必须在 input 层之后、HID report 之前处理。
